## ПРИНЯТЫЕ РЕШЕНИЯ (дизайн)

1. **Единый список наблюдения** (вместо двух списков «чекер» + «личный RH-RU»):
   запись = домен + тумблер «через обход». Чекер проверяет ВСЕ записи; в remote-
   список RH-RU (Worker `/mylist?key=kN`) попадают только записи с включённым
   тумблером. «Удаление ожившего» = выключение тумблера ПО КНОПКЕ ДИАНЫ после
   показа результатов проверки (НЕ авто). Домен остаётся под наблюдением.
   Хранение: store `rh_watch` (источник правды) ↔ KV `mylist:kN` (зеркало,
   только route=on). Стартовое наполнение: ТОЛЬКО whoosh.bike (вкл) —
   контрольные НЕ добавляем (решение Дианы).
2. **Каналы данных**: удалённые — прямой fetch `/dashboard` (live);
   локальные (список, pending, runlog, срез getConfig) — маршрут диспетчера
   `GET rh.box/local`; аварийные — кэш rh_dash + localStorage (как v0.1.0).
3. **Маршруты диспетчера routehub-dash.js v0.2.0**: `/` (HTML), `/local`,
   `/add?d=`, `/del?d=`, `/toggle?d=`, `/sync`, `/recheck?d=`.
   ВАЖНО (T1): проверка доменов — HTML-fetch под нужным режимом, НЕ DIRECT-проба
   скриптом (node игнорируется — см. ниже).
4. **Мутации списка — только через диспетчер** (store = источник правды,
   $httpClient без CORS). Чтения /dashboard — напрямую с Worker (быстрее).
5. **Применение списка в Loon** (мгновенность): KV — мгновенно; Loon тянет
   Remote Rule по `update-interval` (целевое 60 с — тест T3) + кнопка
   «Применить в Loon» = универсальная ссылка nsloon.com/openloon/update?sub=all
   (тест T2: тянет ли Remote Rule). API записи правил / перезагрузки ресурсов
   в Script API НЕТ — мгновеннее нельзя.
6. **Вход-«приложение»**: (а) PWA standalone — меты apple-mobile-web-app-capable
   + apple-touch-icon (Worker /apple-touch-icon.png); иконка на домашнем экране
   открывает дашборд полноэкранно; вёрстка с env(safe-area-inset-*);
   (б) плагин-«визитка» RouteHub-Dash.plugin: метаданные + `#!homepage=
   http://rh.box` + `#!icon=` Worker /icon.png; скриптов и правил НЕТ.
7. **История РКН — на Worker** (rkn_hist:<kN>, 50/смена; /dashboard отдаёт 20).
   **Runlog — только локально** (rh_runlog, 100 записей; через /local).
8. **CORS-дефект Worker** исправлен в v1.7.0 (ACAO:* + OPTIONS).
9. **Вкладки HTML v0.2.0**: Обзор / Узлы / Домены / История / Система.
   Live-таймер 15 с (10/15/30/пауза) + Page Visibility. Индикатор источника.
   Вкладка «Система»: реестр скриптов (описания вшиты), возрасты кэшей,
   переходы loon://LogLists и loon://requestLists.
10. **Пуши с deep-link**: openUrl=http://rh.box/#вкладка.
11. **НЕВОЗМОЖНОЕ**: $done({node}) для HTTPS требует MITM; пассивный лог
    неудач — нет; **$httpClient НЕ маршрутизирует по node** (T1, см. ниже).
12. recheck блокируется в UI при rkn.mode=whitelist.
13. Авто-триггеры по опросу Worker — ОТЛОЖЕНЫ.

## УРОКИ (зафиксировано)

- **Бинарники (PNG) через текстовые MCP-инструменты заливать НЕЛЬЗЯ** — base64
  искажается/обрывается (iconResp на PNG падал с atob-ошибкой). Иконки —
  только SVG (текст, передаётся надёжно), отдаёт Worker.
- **replace в Worker берёт ПЕРВОЕ вхождение** — дублирующий плейсхолдер
  `# __RH_MYLIST_URL__` в комментарии истории конфига перехватывал подстановку.
  Урок: плейсхолдеры НЕ повторять в комментариях; сверять /config после коммита.
- raw GitHub .conf кэшируется CDN до 5 мин даже с ?cb — ждать обновления.

## ДИЗАЙН-СИСТЕМА (зафиксировано)

Язык «хаб и маршруты», нативный iOS: системный шрифт, нижний таб-бар (5 вкладок,

prefers-color-scheme; иконка приложения — тёмное поле, три+ маршрутные линии
сходятся в точку-хаб, бирюзовый акцент (#04342C / #5DCAA5 / #9FE1CB / #7FE0C0);
скелетоны при загрузке; scale(.98) на нажатии; никаких внешних шрифтов и CDN.
Иконка приложения — SVG, отдаёт Worker (/icon.svg, /icon.png, /apple-touch-icon.png).
