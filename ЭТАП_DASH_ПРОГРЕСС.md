## ПРИНЯТЫЕ РЕШЕНИЯ (дизайн)

1. **Единый список наблюдения** (вместо двух списков «чекер» + «личный RH-RU»):
   запись = домен + тумблер «через обход». Чекер проверяет ВСЕ записи; в remote-
   список RH-RU (Worker `/mylist?key=kN`) попадают только записи с включённым
   тумблером. «Удаление ожившего» = выключение тумблера ПО КНОПКЕ ДИАНЫ после
   показа результатов проверки (НЕ авто). Домен остаётся под наблюдением.
   Хранение: store `rh_watch` (источник правды) ↔ KV `mylist:kN` (зеркало,
   только route=on). Стартовое наполнение: ТОЛЬКО whoosh.bike (вкл) —
   контрольные НЕ добавляем (решение Дианы).
2. **Каналы данных (ПЕРЕСМОТРЕНО в v0.3.0 — bootstrap):** страница на http://rh.box
   НЕ делает XHR. Диспетчер при отдаче HTML САМ собирает всё (локальное из
   $persistentStore + Worker /dashboard через $httpClient в туннеле) и вшивает
   в страницу как __BOOT__. Причина: XHR на https-Worker = mixed-content (Safari
   блокирует); XHR-подзапросы на /local http-request не перехватывает надёжно.
3. **Маршруты диспетчера routehub-dash.js v0.3.0:** `/` (HTML+BOOT), `/add?d=`,
   `/del?d=`, `/toggle?d=`, `/check?d=`, `/sync`. Мутации — через НАВИГАЦИЮ
   верхнего уровня (ссылки/форма на http://rh.box/...), диспетчер делает действие
   и редиректит 303 назад. НЕ через XHR (не перехватывается со страницы).
4. **Источник правды списка — store rh_watch**; KV mylist — зеркало (только ВКЛ).
5. **Применение списка в Loon** (мгновенность): KV — мгновенно; Loon тянет
   Remote Rule по `update-interval` (целевое 60 с — тест T3) + кнопка
   «Применить в Loon» = универсальная ссылка nsloon.com/openloon/update?sub=all
   (тест T2: тянет ли Remote Rule). API записи правил / перезагрузки ресурсов
   в Script API НЕТ — мгновеннее нельзя.
6. **Вход-«приложение»**: (а) PWA standalone — меты apple-mobile-web-app-capable
   + apple-touch-icon (Worker /apple-touch-icon.png); (б) плагин-«визитка»
   RouteHub-Dash.plugin: #!homepage=http://rh.box + #!icon=Worker /icon.png.
7. **История РКН — на Worker** (rkn_hist:<kN>, 50/смена; /dashboard отдаёт 20).
   **Runlog — локально** (rh_runlog, кольцо 50; в BOOT.local.runlog).
8. **CORS-дефект Worker** исправлен в v1.7.0 (ACAO:* + OPTIONS) — но при
   bootstrap CORS не нужен (Worker зовёт диспетчер, не браузер).
9. **Вкладки v0.3.0**: Обзор / Узлы / Домены / История / Система. Обновление —
   reload страницы (кнопка «сейчас» / таймер 15/30с), не XHR.
10. **Пуши с deep-link**: openUrl=http://rh.box/#вкладка.
11. **node в $httpClient — см. T6 (РЕШЕНО): работает для ИМЁН УЗЛОВ, НЕ для имён групп.**
12. recheck недостоверен при whitelist (помечается в результате режимом).

## УРОКИ (зафиксировано)

- **node в $httpClient: работает для ИМЁН УЗЛОВ, НЕ для имён групп-политик**
  (T6). Скриптовые пробы «через группу» молча идут по дефолту. Пробить узел —
  только по его полному имени (из getSubPolicies). DIRECT тоже работает.
- **HTTP-страница Loon (http-request) НЕ может: ходить XHR на https (mixed-
  content, Safari режет); делать XHR-подзапросы на свой домен (http-request
  ловит навигацию, не fetch).** Решение — bootstrap (данные в HTML) + мутации
  навигацией с 303-редиректом.
- **Бинарники (PNG) через текстовые MCP-инструменты — НЕЛЬЗЯ** (base64 бьётся).
  Иконки — SVG, отдаёт Worker.
- **LITERAL-СУРРОГАТЫ ЭМОДЗИ (`\uD83C\uDDxx`) ИСКАЖАЮТСЯ при передаче кода текстом**
  (BY `\uD83C\uDDFE` -> `\uDDYE`, CF отклонял деплой). Выносить в именованные
  константы, переиспользовать через [BY]. После коммита кода с эмодзи — node --check.
- **replace берёт ПЕРВОЕ вхождение** — плейсхолдеры не повторять в комментариях.
- raw GitHub .conf кэшируется CDN до 5 мин даже с ?cb.
