## ПРИНЯТЫЕ РЕШЕНИЯ (дизайн)

1. **Единый список наблюдения** (вместо двух списков «чекер» + «личный RH-RU»):
   запись = домен + тумблер «через обход». Чекер проверяет ВСЕ записи; в remote-
   список RH-RU (Worker `/mylist?key=kN`) попадают только записи с включённым
   тумблером. «Удаление ожившего» = выключение тумблера ПО КНОПКЕ ДИАНЫ после
   показа результатов DIRECT-проверки (НЕ авто). Домен остаётся под наблюдением.
   Хранение: store `rh_watch` (источник правды) ↔ KV `mylist:kN` (зеркало,
   только route=on). Стартовое наполнение: whoosh.bike (вкл) + контрольные.
2. **Каналы данных**: удалённые — прямой fetch `/dashboard` (live);
   локальные (список, pending, runlog, срез getConfig) — маршрут диспетчера
   `GET routehub.io/local`; аварийные — кэш rh_dash + localStorage (как v0.1.0).
3. **Маршруты диспетчера routehub-dash.js v0.2.0**: `/` (HTML),
   `/apple-touch-icon*` (PNG-иконка, вшита), `/local`, `/add?d=`, `/del?d=`,
   `/toggle?d=`, `/sync`, `/recheck?d=` (DIRECT-проба ОДНОГО домена; дашборд
   итерирует список сам — каждый перехват укладывается в секунды).
4. **Мутации списка — только через диспетчер** (store = источник правды,
   $httpClient без CORS). Чтения /dashboard — напрямую с Worker (быстрее).
5. **Применение списка в Loon** (мгновенность): KV — мгновенно; Loon тянет
   Remote Rule по `update-interval` (целевое 60 с — тест T3) + кнопка
   «Применить в Loon» = универсальная ссылка nsloon.com/openloon/update?sub=all
   (тест T2: тянет ли Remote Rule). API записи правил / перезагрузки ресурсов
   в Script API НЕТ (перечень API в доке исчерпывающий) — мгновеннее нельзя.
6. **Вход-«приложение»**: (а) PWA standalone — меты apple-mobile-web-app-capable
   + apple-touch-icon, диспетчер отдаёт PNG; иконка на домашнем экране открывает
   дашборд полноэкранно без Safari-хрома; вёрстка с env(safe-area-inset-*);
   (б) плагин-«визитка» RouteHub-Dash.plugin: только метаданные + `#!homepage=
   http://routehub.io` (переход со страницы плагина — дока 7.1); скриптов и
   правил в плагине НЕТ (инъекция argument остаётся в главном конфиге).
7. **История РКН — на Worker**: POST /rkn ведёт кольцевой буфер rkn_hist:<kN>
   (50 записей, только при смене режима); /dashboard отдаёт последние 20.
   **Runlog — только локально**: скрипты пишут кольцевой буфер rh_runlog
   (100 записей {script, ts, ok, note}); дашборд читает через /local.
   На Worker runlog НЕ шлём (решено).
8. **CORS-дефект Worker**: страница http://routehub.io не могла читать JSON
   с workers.dev (нет Access-Control-Allow-Origin) — live-режим v0.1.0 по коду
   работать не должен. Исправлено в worker v1.7.0 (ACAO:* + OPTIONS).
9. **Вкладки HTML v0.2.0**: Обзор / Узлы / Домены / История / Система.
   Live-таймер 15 с (выбор 10/15/30/пауза) + Page Visibility (вкладка скрыта —
   таймер стоит). Индикатор источника: live / кэш / localStorage.
   Вкладка «Система»: реестр скриптов (описания вшиты в HTML), возрасты кэшей,
   переходы loon://LogLists и loon://requestLists.
10. **Пуши с deep-link**: $notification.post c openUrl=http://routehub.io/#вкладка
    (например, rkn при whitelist -> вкладка «Домены»).
11. **Зафиксировано как НЕВОЗМОЖНОЕ**: $done({node}) в http-request для HTTPS
    требует MITM (динамическая маршрутизация скриптом отпадает); пассивный лог
    неудач — нет (подтверждено ранее, Этап F).
12. recheck блокируется в UI при rkn.mode=whitelist (не-whitelist домены на
    DIRECT мертвы по определению — проверка дала бы ложные «не ожил»).
13. Авто-триггеры по опросу Worker — ОТЛОЖЕНЫ (нет действия, которому нужны).

## ДИЗАЙН-СИСТЕМА (зафиксировано)

Язык «хаб и маршруты», нативный iOS: системный шрифт, нижний таб-бар (5 вкладок,
свои контурные иконки — единое семейство, штрих 1.75 pt, скруглённые концы,
inline-SVG); балл узла — кольцевой индикатор (>=70 зелёная зона, 40-69 янтарная,
<40 красная, мёртвый — серое кольцо); режим РКН — баннер под шапкой; индикатор
источника данных — цветная точка (live/кэш/localStorage); тёмная/светлая тема —
prefers-color-scheme; иконка приложения — тёмное поле, три маршрутные линии
сходятся в точку-хаб, бирюзовый акцент (#04342C / #5DCAA5 / #9FE1CB);
скелетоны при загрузке; scale(.98) на нажатии; никаких внешних шрифтов и CDN.
