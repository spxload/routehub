## A.1 — ИНВЕНТАРИЗАЦИЯ ПОДПИСКИ

Подписка: LastdepVPN. Интервал обновления: 12 ч.
Тариф (из announce): 159₽ — 5 устройств; `[VPN]` безлимит,
`[Обход]` — по трафику.

### Трафик и срок (заголовок subscription-userinfo)
- Квота: 70.47 ГиБ. Использовано: 61.79 ГиБ.
- **Остаток: 8.69 ГиБ (12.3 %).**
- **Истекает: 2026-06-03** (≈5 дней от даты сверки).
- upload=0 — учитывается только download.
- Примечание: квота/срок, судя по тарифу, относятся к трафику
  `[Обход]`; `[VPN]` заявлен безлимитным. Точную привязку
  подтвердить у провайдера. Вывод для проекта: экономия
  обходного трафика (🙏) критична — остаток мал.

### Состав: 74 узла, разбивка по тегам в имени
| Тег в имени | Кол-во | Назначение | Транспорт |
|---|---|---|---|
| `[VPN]` (безлимит) | 57 | основной трафик | vless/tls, tcp (xtls-rprx-vision), часть grpc |
| `[Обход]` + `[Обход - МТС]` (🙏) | 12 | платный обход по трафику | vless/tls, ws:443, sni=cdn.deploy-assure.ru |
| `[Игры]` (🕹️) | 5 | низкая задержка | vless/tls, tcp |

### Протоколы (важно для серверного тестировщика)
- security: **tls — 69, reality — 5**. Hysteria2/Wireguard —
  НЕТ. Следовательно ограничение И7 (Xray не протестирует
  Hysteria2/Wireguard) на эту подписку НЕ влияет: все узлы —
  vless, Xray тестирует все.
- transport type: tcp — 58, ws — 12 (все 🙏 Обход), grpc — 4.
- Reality-узлы (5): на `secget.ru` / `de-ru-cache.secget.ru`,
  flow=xtls-rprx-vision.

### Маркеры имён (для regex-фильтров, A.2)
Формат: `<флаг-эмодзи> <категория-эмодзи> <Страна> [<Тег>]`.
Категории-эмодзи: 🙏 Обход, 🚀 reality/быстрый, ⚡/💫/⭐/🟢 VPN,
🕹️ игры. Тег в скобках: `[VPN]`, `[Обход]`, `[Обход - МТС]`,
`[Игры]`. Regex Н15 `^(?!.*?(🙏|Обход)).*` корректно отсекает
обходные узлы — подтверждено на реальных именах.

### КРИТИЧНО: имя узла ≠ реальная гео-локация
Многие узлы делят один входной IP (напр. 23 «страны» из блока
`[Игры]`/одиночных висят на `5.188.156.115`), но несут разные
маршрутные UUID. Реальная страна выхода по имени НЕ определяется.
Подтверждено серверным тестом: узел с именем «🇷🇺 Россия [VPN]
#1» по GeoIP выходит в **FI** (Финляндия). Вывод: маршрутизация
и sticky-логика должны опираться на серверный GeoIP
(`ai-ratings.json` → поле `country`), а не на имя. Это
подтверждает правильность принципа «сервер думает».

### HWID / User-Agent (уточнение)
HWID в перехвате (`64B812C0-…CD18`) совпадает с тем, что уже
зашит в `lastdep-headers.js`. Изменился только билд в User-Agent:
Shadowrocket **3237 → 3274**. Если обновлять скрипт заголовков —
менять нужно UA-билд, HWID прежний.

## СОСТОЯНИЕ СЕРВЕРА (Этап B де-факто готов)

Репозиторий: **`github.com/spxload/Er`** (public, ветка `main`).
Доступ из этого чата подтверждён (чтение/запись — права admin).

Файлы:
- `test_nodes.py` (13.4 КБ) — тестировщик узлов через Xray.
  Скачивает подписку с подменой заголовков, поднимает SOCKS на
  Xray, определяет страну по GeoIP, проверяет 5 ИИ по блок-листу
  стран; при `VERIFY_AI=1` — глубокая проверка эндпоинтов ИИ.
- `.github/workflows/check.yml` — Actions: cron `0 */2 * * *`
  (каждые 2 ч) + ручной запуск; ставит Xray-core, гоняет тест,
  коммитит результат. Секреты: `SUBSCRIPTION_URL`, `SUB_HWID`;
  переменная `VERIFY_AI`.
- `ai-ratings.json` (21.7 КБ) — результат. Это де-факто
  `routehub-ratings.json` из плана.

`ai-ratings.json` — схема и свежесть:
- `updated_iso`: 2026-05-29T13:03:40Z (свежий), `verified`: true.
- `stats`: { total 74, ok 61, failed 13, countries 20 }.
- `nodes`: словарь по имени узла, элемент:
  `{ ok, country, latency(мс), services{chatgpt|claude|gemini|
  grok|perplexity: pass/…}, verified }`.
- 🙏-узлы Обход в основном `ok:false` (таймауты) — ожидаемо,
  они медленные и по трафику.

### Расхождения существующей реализации с планом v3
(не ошибки — упрощения; согласовать имена/состав)
- Имена: репозиторий `Er` (не `RouteHub`), `test_nodes.py`
  (не `routehub-server.py`), `ai-ratings.json`
  (не `routehub-ratings.json`).
- Скоринг ИИ — по блок-листу стран + опц. проверка эндпоинтов.

  3 метрик и префикса `[Lastdep]` — как было в спецификации
  Этапа B плана. Текущая версия проще (соответствует принципу
  «простота важнее»).
