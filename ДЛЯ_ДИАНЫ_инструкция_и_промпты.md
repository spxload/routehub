## 1. НОВАЯ ИНСТРУКЦИЯ ПРОЕКТА (вставить в настройки проекта)

> Заменить весь текущий текст инструкции проекта на этот.
> Коротко — экономит токены. Весь контекст ушёл в файл `СТАРТ.md`.

```
Проект RouteHub — система маршрутизации трафика в Loon (iOS):
держит AI-сервисы (ChatGPT, Claude, Gemini, Grok, Perplexity) на
стабильных узлах, обходит РКН, экономит платный трафик, звонки,

vless), только iOS.

В НАЧАЛЕ КАЖДОГО ЧАТА: прочитай файл `СТАРТ.md` из файлов проекта.
В нём текущий статус, навигация по файлам и главные выводы.
Главный документ — `ПЛАН_РЕАЛИЗАЦИИ_v3.md`, сверяйся с ним.

СТИЛЬ: академичный русский, без воды и вводных фраз. Только
проверенные факты; нет данных — «не могу подтвердить»; источники

ПРЕДСКАЗУЕМОСТЬ важнее количества функций.

ПРАВИЛО ФАЙЛОВ: чаты не видят переписку друг друга, только файлы
проекта. Решили важное — обновили файл. Один этап = один чат.
```

> Вставь в конце текущего чата, когда этап закончен. Я подготовлю


## 4. КАК ОБНОВЛЯТЬ ФАЙЛЫ ПРОЕКТА (порядок действий)

1. В конце этапа я заливаю обновлённые файлы в репозиторий
   `spxload/routehub` и говорю, какие именно изменились.
2. Ты заходишь в репозиторий, скачиваешь изменённые файлы.
3. В настройках проекта (Файлы проекта) — удаляешь старую версию,
   загружаешь новую.
4. Готово — следующий чат увидит свежие файлы.

> Почему так: новый чат видит ТОЛЬКО «Файлы проекта» и инструкцию.
> Файлы в репозитории он сам НЕ читает (только через запрос к
> GitHub-инструменту в процессе работы). Поэтому актуальные версии
> должны лежать именно в «Файлах проекта».
