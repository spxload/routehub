## ИТОГОВАЯ МОДЕЛЬ (отменён путь iCloud+Shortcuts)
Отменено: локальный iCloud-снимок конфига и Shortcuts-флип `&net=wifi/cell`
(хрупкая фоновая автоматизация). Вместо этого — **одна подписка, оба набора
узлов сразу**:
- Worker `/nodes?key=kN` отдаёт ОБА набора в одном ответе: блок 🛜 (сорт. по
  wifi-скорости) и блок 📱 (сорт. по cell-скорости), затем обход разово.
  Каждый сервер присутствует дважды (🛜/📱), имена различаются меткой — Loon
  их не схлопывает.
- Разделение по сети — МЕТКОЙ в имени; фильтры `RH-Filter-W-*`/`RH-Filter-C-*`
  режут regex по 🛜/📱.
- Каждая функция (AI/АВТО/Звонки) — родитель `select` из двух fallback-детей
  (`-W`/`-C`); `routehub-netwatch.js` при смене сети флипает родителя
  (`setSelectPolicy`). Узел внутри ребёнка выбирает fallback сам.
- Выбор узла делает fallback. Скрипты-селекторы (RH-Core/RH-Health) и кнопки
  (ai-bad/viewer) удалены. Активны только RH-Speed (спидтест) и RH-Net (флип
  по сети + детект whitelist).
- AI — страновые тиеры, которые СТРОИТ Worker динамически по числу узлов
  (DE первой; тай-брейк скорость → близость к DE; одиночные в catch-all;
  RU/BY исключены). В конфиге — плейсхолдеры `# __RH_AI_FILTERS__` /
  `# __RH_AI_GROUPS__`, Worker их заполняет при отдаче `/config`.

Текущее: **worker v0.9.2**, **routehub.conf = C-draft-18** (живой).
Ссылка конфига: `https://routehub.proton4iker.workers.dev/config?key=k1`.

## ЧТО ВОШЛО В C-draft-18 (этот этап + интеграция РФ-DIRECT)
- РФ→DIRECT слой доменными списками forg-lib (`whitelist-domains`,
  `category-ru`, `hxehex` [+Сбербанк], `apple`, `whitelist-ips`) + банки ЦБ
  (`domains_banking.list`) — ВЫШЕ прокси-списков. Доменные правила бьют
  GEOIP/FINAL → РФ-приложения видят прямой РФ-IP, не режут функции.
- `RH-Прямой = select, DIRECT, RH-Обход` — на нём GEOIP,RU / РФ-списки /
  Apple / банки / DeepSeek. Норм. = DIRECT; под whitelist флип на обход.
- `RH-Главный = select, RH-АВТО, DIRECT` — FINAL (дефолт RH-АВТО).
- Apple → DIRECT (группа RH-Apple убрана), `real-ip`/`skip-proxy` сохранены.
- `hijack-dns=*:53`, `domain-reject-mode=DNS`; убран `disable-udp-ports=443`
  (глушил UDP/443 → ломал FaceTime/звонки).
- DNS: Яндекс только plain-резерв (в DoH-пул не добавлять — отравление РКН).
- Исправлен путь banking: `domains_banking.list` (старый `banking.list` не грузился).

## ЗАФИКСИРОВАННЫЕ ФАКТЫ (детект сети/оператора)
1. Контекст между чатами — в файлах.
2. Детект сети — без геолокации: `ssid` → маяки `generate_204` (DIRECT) →
   `ya.ru` (DIRECT, маркер «рунет жив»). Любой Wi-Fi = сеть без whitelist
   (поведение netwatch намеренное).
3. `detectOperator`: основной — Worker `/whoami` (DIRECT, `cf.asOrganization`);
   запасной — `ECHO_URL` (`yandex.ru/internet/`) → IP → `$utils.ipaso(ip)`.
4. Worker (`*.workers.dev`) при whitelist НЕДОСТУПЕН → whitelist детектит
   только netwatch на устройстве (маяки).
5. У скрипта НЕТ API перекачать подписку (только select-группы). Loon
   игнорирует inline `update-interval` у `[Remote Proxy]` (проверено).
6. `setSelectPolicy` — рабочий (select); fallback-группой скрипт не управляет.
7. `ssid-trigger` — НЕ добавлять (на всех сетях нужен RULE).
8. Страну узла берём ИЗ ИМЕНИ (флаг/слово), не из GeoIP (`rating.country` врёт).
