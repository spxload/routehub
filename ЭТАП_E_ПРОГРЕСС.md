## АРХИТЕКТУРА AI: два режима через Worker (C-draft-13, worker v0.7.0+)

Один источник `routehub.conf` (select + скрипты); fallback-вариант собирает Worker
ТРАНСФОРМАЦИЕЙ.
- Базовый линк `?key=kN` — СКРИПТОВЫЙ (RH-AI select + RH-Core/Health/Net).
- `?key=kN&ai=fallback` ИЛИ флаг `ai_fallback` — FALLBACK: RH-AI select->fallback
  (+ RH-Filter-Обход последним), RH-Core/Health/Net off. Приоритет: ?ai= > флаг > скрипт.
- Уведомления о смене узла — НАТИВНЫЕ Loon. `routehub-ai-watch.js` не подключён.
- Пострановые непересекающиеся фильтры (DE/NL/CH/FR/GB/Прочее) — группировка в RH-AI;
  набор стран провизорный (уточнить по счётчикам).

**ВАЖНО:** базовый линк — СКРИПТОВЫЙ. Для теста fallback — линк с `&ai=fallback`.

## ПЕР-СЕТЕВЫЕ УЗЛЫ — ЛОКАЛЬНЫЙ КОНФИГ + URL-СВИТЧ (worker v0.8.2) — ПОДТВЕРЖДЕНО ВРУЧНУЮ

**Стена (подтверждена):** Loon НЕ обновляет подписку по inline `update-interval`
(проверено: `nodes_n` не растёт) и не по событию смены сети; скрипт перекачать
подписку/конфиг не может (Script API — только select-группы). `loon://update`
открывает приложение — отклонено.

**Находка (Диана):** локальный конфиг в **iCloud** Loon перечитывает СРАЗУ при
изменении файла; при ИЗМЕНЁННОМ URL подписки узлы перекачиваются немедленно,
**без открытия Loon, без обрыва соединений**.

**ПОДТВЕРЖДЕНО ВРУЧНУЮ (2026-06-07):** правка `&net=wifi`↔`&net=cell` в строке
`Lastdep` локального iCloud-конфига → мгновенная перекачка; метки И ИКОНКИ
переключаются на нужную сеть; соединения и выбор RH-AI не рвутся.

**Worker v0.8.2:** `/nodes?key=kN&net=wifi|cell` — рендер узлов под сеть из URL
(иначе `reg.net` от netwatch, иначе wifi): метка ТОЛЬКО этой сети (🛜/📱) +
сортировка по её скорости; `Cache-Control: no-store`.

**Правила пути:**
- Менять ТОЛЬКО `&net=` в URL; имя подписки оставлять `Lastdep` (к нему привязаны
  фильтры `RH-Filter-*`).
- Конфиг = локальный СНИМОК в iCloud. Узлы и скрипты внутри тянутся с Worker/raw —
  обновляются как раньше; структурные правки самого конфига пересохранять файлом.
- Снимок брать со СКРИПТОВОГО линка (`?key=k1&src=worker`, без `&ai=fallback`),
  чтобы работали AI-селектор и netwatch. В строку `Lastdep` добавить `&net=wifi`
  (стартовое значение, которое Shortcuts будет флипать).
- Покрытие сотовой: метки на сотовой только у узлов, измеренных на сотовой
  (BATCH_CELL=5); для всех — флаг `cell_unlim`.

**ОСТАЛОСЬ (последний шаг):** автоматизировать смену `&net=` при смене сети —
Shortcuts-автоматизация (чтение файла → замена `net=wifi`↔`net=cell` → запись),
Loon НЕ открывается. Проверить **надёжность фоновой автоматизации Shortcuts на
смену сети** (слабое звено; план §18.1 такие триггеры держал под подозрением).

**Альтернатива, если Shortcuts ненадёжен:** скрипт-селектор для `RH-АВТО`
(`select` + выбор быстрейшего узла текущей сети по локальной скорости через
`setSelectPolicy`) — мгновенно, без файла и перекачки, но БЕЗ пер-сетевых МЕТОК
(только выбор узла). AI уже выбирается пер-сетево скриптом.

## ЗАФИКСИРОВАННЫЕ ФАКТЫ (детект сети/оператора, доки 6.2)
1. Контекст между чатами — в файлах.
2. Детект сети — БЕЗ геолокации (`ssid` + маяки; whitelist — `ya.ru` жив).
3. `detectOperator`: основной — Worker `/whoami` (DIRECT, `cf.asOrganization`);
   запасной — `ECHO_URL` → IP → `$utils.ipaso(ip)`.
4. Worker (`*.workers.dev`) при whitelist НЕДОСТУПЕН → whitelist детектит ТОЛЬКО
   netwatch на устройстве (маяки).
5. У скрипта НЕТ API перекачать подписку (только интервал/`loon://update`/ручное).
6. `setSelectPolicy` — рабочий (select); fallback-группой скрипт НЕ управляет.
7. Loon ИГНОРИРУЕТ inline `update-interval` у `[Remote Proxy]` (проверено).
8. Локальный iCloud-конфиг перечитывается сразу при изменении файла; смена URL
   подписки → немедленная перекачка узлов без открытия Loon (проверено вручную).

## РЕШЕНИЕ ПО ОПЕРАТОРУ В WHITELIST
Локально `$utils.ipaso(ip)`. Схему «IP → Worker через узел» не строим.
