## Флаги профиля (devices.json у ключа; Worker САМ дописывает = false; правишь в гисте)
- `cell_unlim` — сотовая мерит ВСЕ узлы (а не 5).
- `ewma` — сглаживание скорости/отклика (a=0.6 новое / 0.4 старое). РЕКОМЕНДОВАНО
  для сотовой (она сильно скачет по времени/месту/сигналу).
- `show_rtt` — показывать rtt в имени (`🛜150↓ 100ms`).
- `auto_refresh` — авто-обновление меток через Shortcuts (см. ниже, D.7).
Передача: `cell_unlim`/`ewma` → opts аргумента; `show_rtt`/`auto_refresh`
читает Worker/netwatch. Поля переживают записи Worker; добавляются при /config и /speed.

## ГРАБЛИ (подтверждено — не наступать заново)
1. **Узлы — base64**. Worker atob/btoa.
2. **Loon нормализует пробелы в именах** → матч по `norm`.
3. **`getSubPolicies(group, cb)` → JSON-СТРОКА**, парсить; имя в `.name`.
4. **`__down?bytes=0` → "Empty response data"**. Мерить `bytes=1`.
5. **`ssid` из `getConfig()` часто пуст** → детект сети по ssid ненадёжен.
   НО детект возможен БЕЗ геолокации — см. «Детект сети без геолокации».
6. **Loon кэширует remote-скрипты по URL** → `?v=<ts>` (ВРЕМЕННО, D.9).
7. **Версия в логе** (`var VERSION`).
8. **rtt — НЕ пинг** (TCP+TLS), min из 3 проб.
9. **Loon НЕ даёт скрипту менять URL подписки / перезагружать конфиг** (офсайт).
   Обновить подписку — только `loon://update?sub=all` (тап/Shortcuts). «2 подписки,
   скрипт переключает» — НЕВОЗМОЖНО. Обход — обе метки + смена УЗЛА скриптом.
10. **Иконки верны только при верном детекте сети.** `🛜`=Wi-Fi, `📱`=сотовая.
    Без геолокации ssid пуст → нужен IP-метод детекта, иначе wifi уедет в `📱`.
11. **Worker рендерит nodes-kN из MERGE (metrics-kN.json), НЕ с нуля.** Иначе POST
    одной сети затирал метки другой (был баг v≤0.4.14: сотовый прогон с пустым
    wifi-кэшем стирал все `🛜`). Теперь POST обновляет только присланные сети,
    вторая сохраняется. Исправлено в v0.4.15.

## Метод замера скорости (почему НИЖЕ Ookla/RT) — НЕ баг
- Наш замер = `speed.cloudflare.com` ЧЕРЕЗ VPN-узел, ОДИН поток. Ookla/RT = «сырой»
  канал до БЛИЖНЕГО сервера РФ, БЕЗ VPN, МНОГО потоков → у них цифры выше.
- Один TCP-поток через канал с RTT ~100–180 мс упирается в «окно/RTT» (BDP), поэтому
  абсолют занижен. Наши 3–4 Mbps совпадают с приложением Cloudflare — замер консистентен.
- Для НАШЕЙ задачи это правильно: нужен не максимум, а СРАВНЕНИЕ узлов одним методом
  + учёт задержки. Для ранжирования валидно. «Реалистичные» цифры (как Ookla) — только
  если добавить параллельные потоки (больше трафика), для выбора узла не требуется.

## Только «due» узлы мерятся (НЕ все каждый раз)
Меряются лишь узлы, которым «пора»: никогда не мерянные / старше 24 ч / на повторе.
Свежие (≤24 ч) пропускаются (батарея/трафик). cell_unlim расширяет ТОЛЬКО размер
батча (80), но НЕ форсирует перемер свежих. Полное покрытие набирается за прогоны.

## Детект сети БЕЗ геолокации (СЛОЁНЫЙ, ключевой дизайн D.7)
Нужен ТОЛЬКО реальный выходной IP сети доступа; тип Loon отдаёт лишь через `ssid`
(геолокация), сервер видит IP УЗЛА. Берём первый сработавший слой:
1. **Быстрый:** `ssid` (если дана геолокация) — мгновенно.
2. **Обычный:** прямой `/whoami` (`node:"DIRECT"`) → Worker читает
   `request.cf.asn`/`asOrganization` → `{net}`. Классификатор `CELL_HINTS`.
3. **Под белым списком / Cloudflare недоступен напрямую:** прямой запрос к IP-эху
   на домене, доступном под whitelist (российский/разрешённый) → реальный IP →
   классификация ЛОКАЛЬНО `$utils.ipaso(ip)`, ЛИБО IP на Worker ЧЕРЕЗ УЗЕЛ.
**Автообучение «дома»:** ssid доступен → запомнить `rh_home_aso`; ssid пуст →
оператор==`rh_home_aso` → Wi-Fi, иначе сотовая.
**ОТКРЫТО (не выдумывать URL):** подобрать/проверить IP-эхо под whitelist (кандидаты
2ip/check-host из `domains_geo_detect`; чистый IP/JSON + прямая доступность).
**Оговорка:** под полным whitelist спидтест ПРОПУСКАЕТСЯ, маршрут держит нативный
fallback на обход → детект тогда некритичен.
API: `$utils.ipaso/ipasn`, `node:"DIRECT"`, `request.cf` (все планы CF).

## Авто-обновление меток через Shortcuts (D.7, флаг auto_refresh)
- Триггер «Письмо» в Shortcuts ЕСТЬ и на новых iOS запускается без тапа. Схема:
  netwatch→Worker→письмо с кодовым словом→Email-автоматизация→`loon://update?sub=all`.
- Ограничение: Apple ПРИНУДИТЕЛЬНО показывает баннер (убрать нельзя) + `loon://update`
  выводит Loon на передний план → «без уведомления» невозможно.
- Тихо: задать подписке `update-interval` (метки сами раз в N часов, не мгновенно).

## Поведение скорости/сети (РЕШЕНО)
- **Обе метки в имени:** `база · 🛜<wifi>↓ 📱<cell>↓` (+` <rtt>ms` при show_rtt).
  Сортировка по Wi-Fi. Мгновенность — на уровне ВЫБОРА узла, не подписи.
- Метки хранятся в metrics-kN.json (MERGE), освежаются при перекачке подписки/auto_refresh.

## Надёжность спидтеста (РЕШЕНО)
- 0/сбой не «готов», хорошее не затирается; повтор `RETRY_MS=15м`.
- После `MAX_FAILS=5` → «мёртв»: бэкофф `DEAD_MS=6ч` + `🛜⛔/📱⛔`.
- На сервер только хорошие (`down>0`) или мёртвые (`dead`). EWMA по флагу.
- Удаление мёртвых — серверная живность (D.6), не телефон.

## Расписание / нативка / РКН
- RH-Speed `cron "0 */20 * * * ?"` весь день; холостые тики дёшевы.
- `ssid-trigger` (режим по сети). РКН live: нативный `fallback` (обычные первыми,
  обход последними). При whitelist спидтест пропускать.

## Что МОЖЕТ скрипт при смене сети (network-changed, D.7)
- Узнать сеть (слоёный детект); перевыбрать узел (мгновенно); сотовый замер;
  setRunningModel; `$notification`; POST на Worker.
- НЕ может: переписать конфиг/URL подписки/имена; перезагрузить подписку молча.
  Только ПЕРВЫЙ network-changed. iOS может пропустить событие.

## Контракты (для D.5+)
- **Аргумент:** `"<key>|<origin>|<opts>"`, opts: `cellall`, `ewma`.
- **POST /speed:** `{key,nonce,wifi:[ITEM],cell:[ITEM]}`, ITEM=`{name,down,rtt}` или
  `{name,dead:true}`. Пустой массив сети = «не обновлять её» (MERGE сохранит старое).
- **GET /whoami:** → `{ip,asn,aso,country,net}` (нужен прямой запрос).
- **Имя:** `база · 🛜<down|⛔>[ rttms] 📱<down|⛔>[ rttms]`. Ключ узла = до ` · `.
- **metrics-kN.json:** `{matchKey:{w:{down,rtt}|{dead},c:{...}}}` — серверное merge-состояние.
- **Кэш телефона:** `rh_speed_wifi`/`rh_speed_cell`: `{baseName:{down,rtt,ts,att,fails}}`
  + `rh_home_aso`.
- **Переключение группы (A.12):** `$config.getConfig(policyName, selectName)`.
