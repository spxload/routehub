## Флаги профиля (devices.json у ключа; правятся в гисте на GitHub; default false)
- `cell_unlim` — сотовая мерит ВСЕ узлы (а не 5). Для безлимитной сотовой.
- `ewma` — сглаживание скорости/отклика (a=0.6 новое / 0.4 старое): один
  случайный замер не дёргает рейтинг.
- `show_rtt` — показывать rtt в имени (`📶150↓ 100ms`); иначе только скорость.
Передача: `cell_unlim`/`ewma` Worker кладёт в opts аргумента (`cellall`,`ewma`);
`show_rtt` Worker применяет сам при рендере имени. Поле переживает записи Worker.

## ГРАБЛИ (подтверждено — не наступать заново)
1. **Узлы — base64** (publish_nodes.py). Worker atob/btoa.
2. **Loon нормализует пробелы в именах** → матч по `norm`. Иначе tested:0.
3. **`getSubPolicies(group, cb)` → JSON-СТРОКА**, парсить. Имя в `.name`.
4. **`__down?bytes=0` → "Empty response data"**. Мерить на `bytes=1`.
5. **`ssid` из `getConfig()` часто пуст** на iOS → детект сети ненадёжен.
6. **Loon кэширует remote-скрипты по URL** → Worker `?v=<ts>` (ВРЕМЕННО, D.9).
7. **Версия в логе** (`var VERSION`) — обязательна.
8. **rtt — НЕ пинг** (TCP+TLS+ответ), выше пинга. min из 3 проб.
9. **Loon НЕ даёт скрипту менять URL подписки / перезагружать конфиг.**
   Сверено по ОФИЦИАЛЬНОМУ сайту nsloon.app: единственный способ обновить
   подписку — URL-схема `loon://update?sub=all` (= `nsloon.com/openloon/update?sub=all`),
   и это действие «в один клик» — ссылку надо ОТКРЫТЬ (тап). Script API метода
   перезагрузки НЕ имеет. Значит «2 подписки, скрипт переключает по сети» —
   НЕВОЗМОЖНО автоматически. Максимум: пуш с `openUrl=loon://update?sub=all`,
   по которому пользователь ТАПАЕТ. Обход — обе метки в имени + смена УЗЛА скриптом.
10. **Иконки 📶/📱 верны только при верном детекте сети.** Иконка = в какой кэш
    (wifi/cell) лёг замер; кэш по ssid (есть→wifi, пусто→cell). Если у Loon НЕТ
    доступа к геолокации — ssid на Wi-Fi не читается → wifi-замер уедет в cell →
    `📱` неверно. ФИКС: дать Loon доступ к геолокации (Always) → ssid читается.

## Поведение скорости/сети (РЕШЕНО)
- **Обе метки в имени:** `база · 📶<wifi>↓ 📱<cell>↓` (при show_rtt + ` <rtt>ms`).
  Телефон шлёт ОБА кэша; сортировка nodes-kN по Wi-Fi.
- **Мгновенность — на уровне ВЫБОРА узла** (скрипт `getConfig(имя,выбор)`), не
  подписи. Подписи обновляются только при перекачке подписки Loon (интервал).
- **Переотправки нет** (POST только после замера). rtt в имя — по флагу show_rtt.

## Надёжность спидтеста (РЕШЕНО)
- 0/сбой не «готов», хорошее значение не затирается; повтор по `RETRY_MS=15м`.
- После `MAX_FAILS=5` неудач → «мёртв»: бэкофф `DEAD_MS=6ч` + маркер `📶⛔/📱⛔`.
- На сервер только хорошие (`down>0`) или мёртвые (`dead`).
- **EWMA** (флаг ewma): сглаживание, чтобы один замер не дёргал рейтинг.
- Удаление мёртвых — НЕ телефоном (серверная живность, D.6); телефон метит ⛔.

## Что МОЖЕТ скрипт при смене сети (network-changed, D.7)
- Узнать сеть; перевыбрать узел группы (мгновенно, из кэша); лёгкий сотовый
  замер; setRunningModel; `$notification` (в т.ч. openUrl=loon://update?sub=all).
- НЕ может: переписать конфиг/URL подписки/имена; перезагрузить подписку молча.
  Только ПЕРВЫЙ network-changed скрипт. iOS может пропустить событие.

## Нативная реакция Loon на сеть + РКН-аварийность (Этап F)
- `ssid-trigger` (режим по сети), `[Host]` ssid: (DNS по сети) — без скрипта.
- РКН live: нативный `fallback` (обычные первыми, обход 🙏 последними) → Loon
  уводит на обход за секунды. Cron-детектор — вторичный слой.

## Спидтест и списки правил — защита (D.5/D.9/F)
- Запрос пинится к узлу. REJECT на `speed.cloudflare.com` теоретически уронит —
  ПРОВЕРИТЬ; защита: allow-правило выше reject; «0 не затирает хорошее» (есть).
- При whitelist обычные дают 0 → спидтест ПРОПУСКАТЬ (Этап F).

## Контракты (для D.5+)
- **Аргумент спидтеста:** `"<key>|<origin>|<opts>"`, opts — через запятую:
  `cellall` (сотовая все), `ewma` (сглаживание).
- **POST /speed:** `{key, nonce, wifi:[ITEM], cell:[ITEM]}`,
  ITEM = `{name,down,rtt}` (хороший) ИЛИ `{name,dead:true}` (нерабочий).
- **Имя:** `база · 📶<down|⛔>[ rttms] 📱<down|⛔>[ rttms]`. Ключ узла = до ` · `.
- **Кэш телефона:** `rh_speed_wifi`/`rh_speed_cell`:
  `{ baseName: {down,rtt,ts,att,fails} }`. ts=last good, att=last попытка,
  fails=неудач подряд (сброс при good).
- **Переключение select-группы (A.12):** `$config.getConfig(policyName, selectName)`.
