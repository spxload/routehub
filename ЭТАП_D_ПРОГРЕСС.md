## ГРАБЛИ (подтверждено — не наступать заново)
1. **Узлы — base64**. Worker atob/btoa.
2. **Loon нормализует пробелы в именах** → матч по `norm`.
3. **`getSubPolicies(group, cb)` → JSON-СТРОКА**, парсить; имя в `.name`.
4. **`__down?bytes=0` → "Empty response data"**. Мерить `bytes=1`.
5. **`ssid` из `getConfig()` часто пуст** → детект сети по ssid ненадёжен.
   НО детект сети возможен БЕЗ геолокации — см. раздел «Детект сети без геолокации».
6. **Loon кэширует remote-скрипты по URL** → `?v=<ts>` (ВРЕМЕННО, D.9).
7. **Версия в логе** (`var VERSION`).
8. **rtt — НЕ пинг** (TCP+TLS), min из 3 проб.
9. **Loon НЕ даёт скрипту менять URL подписки / перезагружать конфиг** (сверено
   по офсайту nsloon.app). Обновить подписку — только URL-схема `loon://update?sub=all`
   (открыть тапом/через Shortcuts). Script API метода нет. «2 подписки, скрипт
   переключает» — НЕВОЗМОЖНО автоматически. Обход — обе метки + смена УЗЛА скриптом.
10. **Иконки верны только при верном детекте сети.** `🛜`=Wi-Fi, `📱`=сотовая.
    Без геолокации ssid пуст → нужен IP-метод детекта (ниже), иначе wifi уедет в `📱`.

## Детект сети БЕЗ геолокации (ключевой дизайн D.7)
Проблема была НЕ «отправить на сервер» (скрипт шлёт что угодно), а «узнать ТИП сети».
Loon отдаёт скрипту тип только через `ssid` (нужна геолокация). Сервер тоже не видит —
он видит выходной IP УЗЛА (VPN), не сеть доступа. Решение без геолокации:
- На смене сети (или в спидтесте) скрипт делает ПРЯМОЙ запрос мимо туннеля:
  `$httpClient.get({url:<ip-эхо>, node:"DIRECT"})` → получает РЕАЛЬНЫЙ выходной IP
  сети доступа (Wi-Fi или сотовой).
- Классификация оператора: локально `$utils.ipaso(ip)`/`$utils.ipasn(ip)` ЛИБО
  отправить IP на сервер (там MaxMind). Мобильный оператор (МТС/Мегафон/Билайн/Tele2)
  → сотовая; домашний провайдер → Wi-Fi.
- Автообучение «дома»: когда `ssid` доступен (точно Wi-Fi) — один раз запомнить
  домашний ASO в `rh_home_aso`; когда `ssid` пуст — оператор==`rh_home_aso` → Wi-Fi,
  иначе → сотовая. Деградация плавная: с геолокацией — точно; без — по IP-методу.
- Стоимость: один маленький прямой запрос на смену сети. API документированы
  (`$utils.ipaso/ipasn`, `node:"DIRECT"`) — подтвердить поведение на устройстве.
Геолокация остаётся лишь необязательным быстрым путём. Реализация — D.7 (netwatch);
тот же детект использует спидтест для выбора кэша wifi/cell.

## Авто-обновление меток подписки через Shortcuts (D.7, флаг auto_refresh)
Цель: метки в Loon освежаются без ручной перекачки. Проверено по Apple Support:
- **Триггер «Письмо» (Email) в Shortcuts существует** (Sender / Subject Contains /
  Account / Recipient) и на современных iOS **запускается сразу, без тапа**.
  Схема: netwatch шлёт на Worker «сменилась сеть» → Worker шлёт письмо с кодовым
  словом → Email-автоматизация ловит → открывает `loon://update?sub=all`.
- **Ограничение (неустранимое):** Apple ПРИНУДИТЕЛЬНО показывает баннер при
  авто-запуске таких автоматизаций (тумблер «Notify When Run» в Run Immediately
  убрать нельзя). Плюс открытие `loon://update` выводит Loon на передний план.
  → «полностью бесшовно без уведомления» невозможно; будет короткий баннер + Loon.
- Альтернатива без баннеров: задать подписке `update-interval`, метки сами
  освежаются раз в N часов (тихо, но не мгновенно).
- Также есть прямой триггер Wi-Fi (connect) — без письма/Worker, но только в одну
  сторону (подключение к Wi-Fi); письмо-мост покрывает обе стороны.
Реализация (письмо из Worker/сигнал из netwatch) — Этап D.7, по флагу auto_refresh.

## Поведение скорости/сети (РЕШЕНО)
- **Обе метки в имени:** `база · 🛜<wifi>↓ 📱<cell>↓` (+` <rtt>ms` при show_rtt).
  Сортировка по Wi-Fi. Мгновенность — на уровне ВЫБОРА узла, не подписи.
- Подписи освежаются при перекачке подписки Loon (интервал) либо auto_refresh.
- Переотправки нет (POST только после замера).

## Надёжность спидтеста (РЕШЕНО)
- 0/сбой не «готов», хорошее не затирается; повтор `RETRY_MS=15м`.
- После `MAX_FAILS=5` → «мёртв»: бэкофф `DEAD_MS=6ч` + `🛜⛔/📱⛔`.
- На сервер только хорошие (`down>0`) или мёртвые (`dead`). EWMA по флагу.
- Удаление мёртвых — серверная живность (D.6), не телефон.

## Расписание (routehub.conf [Script])

## Что МОЖЕТ скрипт при смене сети (network-changed, D.7)
- Узнать сеть (по IP-методу, без геолокации); перевыбрать узел (мгновенно);
  лёгкий сотовый замер; setRunningModel; `$notification` (openUrl);
  POST на Worker (сигнал auto_refresh / выходной IP для классификации).
- НЕ может: переписать конфиг/URL подписки/имена; перезагрузить подписку молча.
  Только ПЕРВЫЙ network-changed. iOS может пропустить событие.

## Контракты (для D.5+)
- **Аргумент:** `"<key>|<origin>|<opts>"`, opts через запятую: `cellall`, `ewma`.
- **POST /speed:** `{key,nonce,wifi:[ITEM],cell:[ITEM]}`,
  ITEM=`{name,down,rtt}` или `{name,dead:true}`.
- **Имя:** `база · 🛜<down|⛔>[ rttms] 📱<down|⛔>[ rttms]`. Ключ узла = до ` · `.
- **Кэш телефона:** `rh_speed_wifi`/`rh_speed_cell`: `{baseName:{down,rtt,ts,att,fails}}`.
  Плюс `rh_home_aso` (домашний оператор для детекта сети).
- **Переключение группы (A.12):** `$config.getConfig(policyName, selectName)`.
