## ГРАБЛИ (подтверждено — не наступать заново)
1. **Узлы — base64** (publish_nodes.py). Worker atob/btoa.
2. **Loon нормализует пробелы в именах** → матч по
   `norm = s.replace(/\s+/g,' ').trim()`. Иначе tested:0.
3. **`getSubPolicies(group, cb)` отдаёт JSON-СТРОКУ**, не массив.
   Парсить. Элементы `{"type":"node","name":"..."}`; имя в `.name`.
4. **Проба `__down?bytes=0` → "Empty response data"** (пустое тело).
   Мерить на `bytes=1`.
5. **`ssid` из `getConfig()` часто пуст** на iOS → детект сети по ssid
   ненадёжен; пусто → 'cell'. Надёжный детект — D.7 (netwatch).
6. **Loon кэширует remote-скрипты по URL** → Worker добавляет `?v=<ts>`
   к script-path (ВРЕМЕННО, снять в D.9). Задержка доставки ~3–5 мин.
7. **Версия в логе** (`var VERSION`) — обязательна, бамп при правке.
8. **rtt — НЕ пинг.** Время полного запроса через узел (TCP+TLS+ответ),
   всегда выше пинга, первый «холодный» даёт всплеск. Берём min из 3 проб.
9. **Loon НЕ даёт скрипту менять URL подписки / подгружать другой конфиг.**
   Поэтому «2 подписки (wifi/cell), скрипт переключает по сети» — НЕВОЗМОЖНО.
   Обход — обе метки в одном имени + выбор узла скриптом (см. ниже).

## Поведение скорости/сети (РЕШЕНО)
- **Обе метки в имени.** `база · 📶<wifi>↓ 📱<cell>↓` (что измерено). Снимает
  «мгновенное переключение» — обе цифры всегда видны, переворачивать не нужно.
  Телефон шлёт ОБА кэша (wifi+cell), Worker вписывает обе.
- **Сортировка nodes-kN — по Wi-Fi.** Правильный узел под ТЕКУЩУЮ сеть выбирает
  скрипт-селектор (D.5) из локального кэша нужной сети — «мгновенность» на
  уровне ВЫБОРА, не подписи.
- **Имена обновляются только при перекачке подписки** Loon (по интервалу),
  не мгновенно — предел Loon (имена неизменяемы скриптом). Мгновенный просмотр
  текущих скоростей — задача `routehub-viewer.js`.
- **Переотправка убрана.** POST только после реального замера.
- **rtt в имя НЕ пишем** (раздувает); хранится в кэше для выбора и viewer.

## Надёжность спидтеста (лимит повторов, dead-маркер) — РЕШЕНО
- **0/сбой не кэшируется как «готово».** Хорошее значение нулём не затирается.
- **Сбойный перемеряется по бэкоффу** `RETRY_MS=15м`, пока не даст `down>0`.
- **Лимит повторов:** после `MAX_FAILS=5` неудач узел = «мёртв» → длинный
  бэкофф `DEAD_MS=6ч` (не долбить весь день, экономия батареи/трафика).
- **Статус вместо скорости:** мёртвый узел шлётся как `{name,dead:true}` →
  в имени `📶⛔`/`📱⛔` (по каждой сети отдельно: может быть `📶150↓ 📱⛔`).
- **На сервер уходят только** хорошие (`down>0`) ИЛИ мёртвые (`dead`);
  «в процессе» (неудач < MAX) — нет.
- **Удаление мёртвых — НЕ телефоном.** Спидтест-смерть = «не скачалось через
  узел в этой сети» (может быть сетевым/временным). Реальное удаление —
  серверная живость (`routehub-ratings.json`, D.6/сервер) либо узел сам
  исчезает из подписки провайдера. Телефон только метит `⛔` + отступает.
- **Батарея:** холостой тик (мерить нечего) — доли секунды без сети; расход
  только от самих замеров, а они ограничены 24ч-кэшем + лимитом повторов.

## Расписание (routehub.conf [Script])
- RH-Speed: `cron "0 */20 * * * ?"` — ВЕСЬ ДЕНЬ каждые 20 мин. Днём (сотовая на
  работе) мерит сотовую + повторяет сбойные; вечером/ночью дома — полный
  Wi-Fi-свип (когда узлы «несвежие»). Когда всё готово — тики вхолостую.
- RH-Core 30м, RH-Health 20м, RH-Net (network-changed). Все enabled=false в
  базе; RH-Speed на время D форс-включает Worker.

## Что МОЖЕТ скрипт при смене сети (network-changed, D.7)
- Узнать сеть (оговорка ssid, грабли №5), писать флаг в store.
- **Перевыбрать узел** select-группы под новую сеть (мгновенно, из кэша).
- Запустить лёгкий сотовый замер. Переключить режим Loon. Послать `$notification`.
- Делать запросы через узлы (`$httpClient ... node`).
НЕ может: переписать конфиг/фильтры/группы/URL подписки/имена узлов; сменить
сеть; подгрузить другой конфиг (грабли №9). Только ПЕРВЫЙ network-changed
скрипт срабатывает → один диспетчер. iOS может пропустить событие при выгрузке.

## Нативная реакция Loon на сеть (без скрипта)
- `ssid-trigger` в `[General]`: режим по сети (`"cellular":PROXY,"default":RULE`);
  `cellular`/`default` по ТИПУ сети (SSID не нужен). `[Host]` `ssid:` — DNS по сети.
- Меняет только РЕЖИМ и DNS; пересобрать/переранжировать узлы Loon сам не может.

## РКН-аварийность — НАТИВНО, не скриптом (Этап F)
- «Live»-переход на обход — нативный `fallback`: обычные ПЕРВЫМИ, обход 🙏
  ПОСЛЕДНИМИ. Loon уводит на обход за секунды, без ожидания cron.
- Обход включается только когда все обычные мертвы (= whitelist). Тюнить
  interval/tolerance. Cron-детектор РКН — вторичный слой.

## Спидтест и списки правил — защита (D.5/D.9/F)
- Запрос пинится к узлу (`node:`) — правила обычно не перенаправляют. НО
  `REJECT` на домен теста (`speed.cloudflare.com`) теоретически уронит → 0.
  **ПРОВЕРИТЬ на устройстве** при включённых списках.
- Защита: allow-правило `speed.cloudflare.com` ВЫШЕ reject-списков; в спидтесте
  «0 не затирает хорошее» (уже сделано v0.4.10+).
- При реальном whitelist обычные честно дают 0 → спидтест ПРОПУСКАТЬ (Этап F).

## Контракты (для D.5+)
- **Аргумент спидтеста:** `"<key>|<origin>|<opts>"` (opts=`cellall` при cell_unlim).
- **POST /speed:** `{key, nonce, wifi:[ITEM], cell:[ITEM]}`, где
  ITEM = `{name,down,rtt}` (хороший) ИЛИ `{name,dead:true}` (нерабочий).
- **Метрика в имени:** `база · 📶<down|⛔> 📱<down|⛔>` (что есть).
  Стабильный ключ узла = имя до первого ` · ` (`baseName/stripMetric`).
- **Локальный кэш:** `rh_speed_wifi`, `rh_speed_cell` в `$persistentStore`:
  `{ baseName: {down, rtt, ts, att, fails} }`. ts=последний good, att=последняя
  попытка, fails=неудач подряд (сброс при good).
- **Переключение select-группы (A.12):** `$config.getConfig(policyName, selectName)`
  (true/false). Подтвердить на устройстве при первом запуске core.
