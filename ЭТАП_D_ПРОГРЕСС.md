## ГРАБЛИ (не наступать)
1. Узлы — base64. 2. Loon нормализует пробелы → матч по `norm`. 3. `getSubPolicies`→
JSON-СТРОКА, имя в `.name`. 4. `__down?bytes=0`→"Empty response"; мерить `bytes=1`.
5. `ssid` часто пуст → детект сети не по ssid (см. формула/детект, D.7). 6. Loon кэширует
скрипты по URL → `?v=<ts>` (ВРЕМЕННО, D.9). 7. VERSION в логе. 8. rtt — не пинг (min из 3).
9. Loon НЕ даёт скрипту менять URL подписки/перезагружать конфиг (офсайт); обновление —
только `loon://update?sub=all` тапом/Shortcuts. 10. Иконки сети верны лишь при верном
детекте. 11. **Worker рендерит nodes-kN из MERGE (metrics-kN.json), не с нуля** — POST
одной сети не затирает метки другой (исправлено v0.4.15).

## Оценка узлов — см. ЭТАП_D_ФОРМУЛА.md (кратко)
- **АБСОЛЮТНЫЕ пороги, НЕ относительно соседей** (ключевая правка): `s=min(1,down/20)`
  и т.п. Иначе 100/25/3 → 25 и 3 в одном тире, хотя 25 для видео хватает. Потолок «с
  запасом»: 100 и 25 оба максимум, 3 — низко. Побочно честно гасит «палки».
- **Тиры (видимые)** — достаточность скорости, 3-сегментная полоска `▰▰▰/▰▰▱/▰▱▱/▱▱▱`,
  мёртвый `⛔`. В имени `🛜▰▰▱ 📱▰▱▱`. Цифры — за `show_rtt`. (Worker v0.4.16.)
- **Метрики:** down, rtt, jit(=max−min проб), bl(=отклик под нагрузкой−rtt, approx).
  (Спидтест v0.4.13.)
- **Балл/выбор (D.5):** гейт сервера (живой/страна/UDP) → балл по профилю группы
  (AI / АВТО / Звонки, веса в спеке) → sticky/hysteresis. Рейтинг сервера — ГЕЙТ, не слагаемое.
- **3 профиля** хватает; отдельные под YouTube/веб не нужны (узел, годный для видео,
  годен и вебу). Делить — только если позже реально понадобится.

## Метод замера (почему ниже Ookla) — НЕ баг
Наш замер = через VPN-узел, ОДИН поток, до Cloudflare; Ookla/RT = сырой канал, без VPN,
много потоков. Один поток через RTT-канал занижен (BDP). Для СРАВНЕНИЯ узлов — верно.
«Только due мерятся»: свежие (≤24ч) пропускаются; cell_unlim расширяет лишь размер батча.

## Поведение / надёжность
- Обе сети в одном имени (merge); мгновенность — на уровне ВЫБОРА узла, не подписи.
- 0/сбой не «готов»; повтор RETRY_MS=15м; после MAX_FAILS=5 → мёртв (DEAD_MS=6ч, ⛔).
- Удаление мёртвых — серверная живность (D.6), не телефон.
- RH-Speed cron весь день `0 */20 * * * ?`; холостые тики дёшевы.

## Детект сети БЕЗ геолокации (D.7) — слоёный
1. ssid (если есть геолокация). 2. прямой `/whoami` → `request.cf.asn/asOrganization`
(нужен `node:"DIRECT"`). 3. под whitelist/без CF: прямой запрос к IP-эху на разрешённом
домене → `$utils.ipaso` локально, либо IP на Worker через узел. Автообучение `rh_home_aso`.
**Кэш по оператору** (D.7): на сотовой данные релевантны текущей симке → ключ по оператору.
ОТКРЫТО: подобрать IP-эхо под whitelist (2ip/check-host — проверить).

## Что МОЖЕТ скрипт при смене сети (D.7)
Узнать сеть (слоёный детект); перевыбрать узел (мгновенно); сотовый замер; setRunningModel;
`$notification`; POST на Worker. НЕ может: менять URL подписки/перезагружать молча. Только
ПЕРВЫЙ network-changed.

## Авто-обновление меток (D.7, auto_refresh) — событийное
netwatch→Worker→письмо с кодовым словом→Email-автоматизация (есть, без тапа)→`loon://update`.
Минус: Apple принудительно показывает баннер + Loon на передний план (без уведомления нельзя).
Тихо: `update-interval` подписки (раз в N часов). Постоянные 10 мин — НЕ делаем (churn, вредит sticky).

## Контракты
- POST /speed: `{key,nonce,wifi:[ITEM],cell:[ITEM]}`, ITEM хороший `{name,down,rtt,jit,bl?}`,
  мёртвый `{name,dead:true}`. Пустой массив сети = не обновлять (merge сохранит).
- GET /whoami: `{ip,asn,aso,country,net}` (нужен прямой запрос).
- Имя: `база · 🛜<тир|⛔>[ down↓rtt] 📱<тир|⛔>[…]`. Ключ узла = до ` · `.
- Кэш телефона: `{baseName:{down,rtt,jit,bl,ts,att,fails}}` + `rh_home_aso`.
- Переключение группы (A.12): `$config.getConfig(policyName, selectName)`.
