## ТЕКУЩИЙ СТАТУС

- **Этап C — ЗАКРЫТ.** Главный конфиг `routehub.conf` собран
  (версия C-draft-6 в репозитории): General, фильтры, группы,
  правила, минимальный набор списков, заглушки скриптов под Этап D.
- **Сделано:** A закрыт; B закрыт (сервер готов); C — конфиг собран.
  Диагностика Gemini завершена.
- **Дальше — Этап D** (скрипты). Порядок: D → H → E → F → I → J → G.
- **Этап D — направление выбрано: ЛИЧНЫЕ подписки.** Скорость видна в
  именах узлов, выбор узла (и AI, и основы) — по СКОРОСТИ + ОТКЛИКУ.
  Реализация: динамический сервер (Worker) отдаёт персональный конфиг по
  keyed-ссылке `…/config?key=kN` и принимает скорость; узлы по ключу в
  одном гисте; реестр `devices.json` (free/bound, запас, защита nonce).
  Токен только на Worker, на телефоне — keyed-ссылка. Детали —
  `ЭТАП_D_ЛИЧНЫЕ_ПОДПИСКИ.md` (группа B). Это пересматривает «остальной
  трафик без скрипта» из вывода 9 (вариант «б»).
- **Подписка — через секретный Gist (НЕ MitM).** Сервер
  (`publish_nodes.py`, шаг в check.yml) декодирует подписку Lastdep
  с правильными заголовками и кладёт полный список узлов в секретный
  Gist. Конфиг берёт узлы ссылкой на Gist → MitM и скрипт заголовков
  на телефоне УБРАНЫ (header-подмена была нестабильна). URL провайдера
  в публичном конфиге отсутствует (только в Secrets).
  Secrets: `SUBSCRIPTION_URL`, `SUB_HWID`, `GIST_TOKEN` (scope gist),
  `GIST_ID`. НЕ возвращать MitM/скрипт заголовков.
  Порядок узлов в гисте: Германия первой, дальше по числу [VPN]-узлов;
  grpc оставлены (Loon их не импортирует → в гисте больше, чем в Loon).
- **Конфиг для семьи (Loon Remote Config), одна ссылка:**
  `https://raw.githubusercontent.com/spxload/routehub/main/routehub.conf`
  (запас, если raw из РФ тормозит — A.5).
  **ПОКА НЕ РАЗДАВАТЬ семье** — черновик, скрипты выключены до Этапа D.
  Сначала тест на своём телефоне. (На Этапе D общую ссылку заменят
  персональные keyed-ссылки — см. ЭТАП_D_ЛИЧНЫЕ_ПОДПИСКИ.md.)
- **Проверить на устройстве (фиксируется на сборке G):** конфиг из
  ссылки грузит узлы из Gist (~71 в Loon без grpc), фильтр `RH-AI`
  ловит ~62 (A.2); доступность `gist.githubusercontent.com` и
  `raw.githubusercontent.com` из РФ (A.5); `disable-udp-ports=443` для
  YouTube (A.7); добавление удалённого конфига в Loon 3.3.9 (A.14).
- **Ключевое решение C:** фильтр узлов — `NameKeyword` по
  инвариантному тегу `[VPN]`/`[Игры]`/`[Обход]`, НЕ regex по эмодзи
  (провайдер часто меняет значки). Правится строкой в GitHub → у всех
  (вкл. жену) подтягивается автоматически. Игровые узлы — в обычный
  трафик и звонки (в звонках — ПЕРВЫМИ); AI — только `[VPN]`.
- **Маркеры lmc999 — ЗАКРЫТО:** не внедряем (AI держим на Германии,
  серверная AI-проверка понижена в весе).
- **Открытый вопрос (план):** `ПЛАН_РЕАЛИЗАЦИИ_v3.md` Раздел 6 (Этап C)

## ГЛАВНЫЕ ВЫВОДЫ (зафиксировано, не пересматривать без причины)

1. **Сервер думает, телефон исполняет.** Тяжёлая проверка (живость,
   гео, тип IP, стабильность) — на сервере раз в 2 ч → JSON-рейтинг.
   Loon читает рейтинг. ВАЖНО (уточнено, вывод 10): скорость/пинг —
   НЕ на сервере, а на телефоне; роль сервера смещена в фон.

2. **AI-доступность без логина = только страна + стабильность.**
   Реальный доступ под аккаунтом сервер НЕ видит (вывод 8). Поэтому
   AI-группу не тестируем, а держим на Германии (вывод 9). ChatGPT —
   по факту (compliance), прочие AI — по стране.

3. **Проблема Gemini — НЕ тип IP (опровергнуто, см. вывод 8).**
   Ранняя гипотеза была «Google банит датацентр/VPN». Диагностика
   80 узлов её опровергла: один тип IP / один ASN дают разный
   результат. Тип IP оставлен как атрибут для фильтров, но в вердикт
   Gemini НЕ заводится.

4. **MaxMind — через зеркало P3TERX/GeoLite.mmdb БЕЗ ключа**
   (регистрация на maxmind.com Диане недоступна). Уже внедрено.

5. **РКН — два типа блокировок:** внутренние (РКН режет наружу →
   Re-filter) и внешние (сервисы сами режут RU-IP → no-russia-
   hosts). Это РАЗНЫЕ списки, нужны оба.

6. **TikTok — БЕЗ MITM.** Semporia/TikTok-Unlock отвергнут (MITM,
   устарел, риск бана). Вместо него — маршрутизация доменов TikTok
   на узел нужной страны без переписывания региона.

7. **Авторизованная проверка AI по узлам отвергнута окончательно**
   (impossible-travel-бан, небезопасно).

8. **Причина сбоев Gemini — репутация выходного IP под аккаунтом**,
   а НЕ тип IP/ASN/страна/DNS. Доказано на 80 узлах (dns 80/80
   honest, все гипотезы опровергнуты). Сервер это анонимно не видит.
   Решение НЕ серверное: AI-группу держать на узлах региона, который
   работает.

9. **AI-группа: Германия (всегда первая) + 2 резервные страны**,
   выбираемые динамически (страны с >3 узлов, по числу узлов, при
   равенстве — по скорости), только основные VPN-узлы (НЕ обход/игры).
   Реализация — одна `select`-группа `RH-AI` (пул из фильтра `[VPN]`) +
   cron-скрипт-селектор (скрипт не меняет состав группы, только выбирает
   узел — ограничение Loon). Узел ВНУТРИ страны — по скорости+отклику.
   (Уточнение: «остальной трафик без скрипта» пересмотрено — см. вывод 12.)

10. **Двухслойная модель.** Сервер (общий фон): живость, стабильность
    за недели, классификация (страна/тип IP/ASN), новые узлы. Телефон
    (локально): скорость/пинг (спидтест ночь Wi-Fi + день сотовая),
    оценки кнопок «не работает». Серверный пинг нерепрезентативен —
    скорость только с телефона, локальна у каждого.

11. **Конфиг — одна ссылка для семьи** (Loon Remote Config): жена/дети
    по ссылке получают всё (узлы, группы, фильтры, скрипты). Правится
    ПРЯМО В GitHub (не выгрузкой с телефона). **Токен GitHub на телефоне
    запрещён** (раздаётся по ссылке → утечёт). Кнопки = generic-скрипты,
    занижение локальное в $persistentStore. Фильтры/группы — в УДАЛЁННОМ
    конфиге (текст `routehub.conf`), не в локальных галочках Loon, иначе
    изменения не разойдутся семье. Узлы — из секретного Gist (см. статус).

12. **Личные подписки (Этап D, согласовано).** Чтобы скорость была видна
    в именах и выбор шёл по скорости+отклику, у каждого устройства — свой
    отсортированный список (скорость локальна, вывод 10). Реализация:
    динамический Worker отдаёт персональный конфиг по `…/config?key=kN`
    (Loon при обновлении дёргает ровно эту ссылку → привязка держится; по
    ОБЩЕЙ ссылке устройство не опознать — IP/UA/cookie не годятся),
    принимает скорость, кладёт узлы по ключу в ОДИН гист, ведёт реестр
    `devices.json` (free/bound + запас + защита nonce от двойной выдачи).
    GitHub-токен только на Worker (вывод 11 соблюдён). Одна вставка
    keyed-ссылки на устройство (Loon не даёт скрипту менять URL подписки).
    Детали и поток — `ЭТАП_D_ЛИЧНЫЕ_ПОДПИСКИ.md`.

Детали решений 8–11 — в `ЭТАП_B_РЕШЕНИЯ.md`; решение 12 — в
`ЭТАП_D_ЛИЧНЫЕ_ПОДПИСКИ.md` (репозиторий).
