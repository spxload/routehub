### Что уже работает (D.1–D.4)
- **Worker** `https://routehub.proton4iker.workers.dev` (Cloudflare Free).
  Привязан к репозиторию (**Workers Builds**): деплоит сам при push в
  `main`. `wrangler.toml` в репо; `GIST_TOKEN` — Secret в дашборде,
  остальные env — в wrangler.toml. Проверка деплоя: Cloudflare MCP
  `workers_list` → `modified_on`.
  - `GET /config?key=kN` — персональный конфиг (узлы = nodes-kN).
  - `POST /speed` — приём скорости с телефона, пересборка nodes-kN.
- **Секретный гист** `b14194982ad9d058c21d393d0f342147`: `lastdep-nodes.txt`
  (эталон, base64), `nodes-kN.txt` (узлы профиля, base64), `devices.json`
  (реестр: free/bound/conflict, nonce, cell_unlim), `debug-kN.json`
  (временная диагностика).
- **Спидтест на телефоне** (`routehub-speedtest.js`, cron): меряет
  отклик+скорость через каждый узел, шлёт на Worker, тот вписывает метку
  ` · N↓ Mms` в имя и сортирует по скорости. Проверено: Wi-Fi ~150,
  сотовая ~4–5 Mbps. `k1` (телефон Дианы) забиндан, `k2` свободен (жене).
- **Тумблер сотовой:** поле `cell_unlim:true/false` у ключа в `devices.json`
  (правится в гисте на GitHub) — при true сотовая мерит все узлы.

### ВНИМАНИЕ — временное (снять на D.9)
- В Worker: `?v=<ts>` к script-path (сбиватель кэша Loon); форс
  `enabled=true` строки RH-Speed; файлы `debug-kN.json`.
- Обновление скриптов доходит до Loon с задержкой ~3–5 мин (raw + кэш).
- У каждого скрипта — `var VERSION` в логе (видно, обновился ли).
- Надёжный детект Wi-Fi/сотовая (ssid из getConfig часто пуст на iOS) —
  решаем в D.7 (netwatch). Пока пусто → метка 'cell'.

## КАК СО МНОЙ РАБОТАТЬ (стиль)

- Академичный русский, без «воды» и вводных фраз.
- Только проверенные факты. Нет данных — «не могу подтвердить».
- Источники не выдумывать. Уточнения — списком.

## ГЛАВНЫЕ ВЫВОДЫ (зафиксировано, не пересматривать без причины)

1. **Сервер думает, телефон исполняет.** Тяжёлая проверка (живость,
   гео, тип IP, стабильность) — на сервере раз в 2 ч → JSON-рейтинг.
   Скорость/пинг — НЕ на сервере, а на телефоне (вывод 10).

2. **AI-доступность без логина = только страна + стабильность.**
   AI-группу не тестируем, держим на Германии (вывод 9).

3. **Проблема Gemini — НЕ тип IP** (опровергнуто диагностикой 80 узлов).

4. **MaxMind — через зеркало P3TERX/GeoLite.mmdb БЕЗ ключа.**

5. **РКН — два типа блокировок:** внутренние (Re-filter) и внешние
   (сервисы режут RU-IP → no-russia-hosts). Нужны оба списка.

6. **TikTok — БЕЗ MITM.** Маршрутизация доменов на узел нужной страны.

7. **Авторизованная проверка AI по узлам отвергнута** (бан, небезопасно).

8. **Причина сбоев Gemini — репутация выходного IP под аккаунтом.**
   Решение: AI-группу держать на работающем регионе (Германия).

9. **AI-группа: Германия (всегда) + 2 резервные страны**, только
   основные VPN-узлы. Одна `select`-группа `RH-AI` + cron-селектор
   (`routehub-core.js`, D.5). Узел внутри страны — по скорости+отклику.

10. **Двухслойная модель.** Сервер: живость, стабильность, классификация,
    новые узлы. Телефон: скорость/пинг (спидтест), кнопки «не работает».

11. **Конфиг — одна ссылка** (через Worker, keyed). **Токен GitHub на
    телефоне запрещён.** Фильтры/группы — в удалённом конфиге.

12. **Личные подписки (Этап D).** Worker отдаёт персональный конфиг по
    `?key=kN`, принимает скорость, кладёт узлы по ключу в гист, ведёт
    `devices.json` (free/bound + nonce). Токен — только на Worker.
    Детали — `ЭТАП_D_ЛИЧНЫЕ_ПОДПИСКИ.md`; реализация/статус —
    `ЭТАП_D_ПРОГРЕСС.md`.
