### Что работает (актуальные версии)
- **Worker** `https://routehub.proton4iker.workers.dev` (**v0.9.2**, авто-деплой при push).
  Эндпоинты: `GET /config?key=kN`, `GET /nodes?key=kN`, `GET /whoami`, `GET /status`,
  `POST /speed`. `/config` — берёт шаблон `routehub.conf`, подставляет динамические
  AI-тиры в плейсхолдеры, переписывает script-path в raw-URL, добавляет
  argument к RH-Speed/RH-Net. `/nodes` — отдаёт ОБА набора (🛜 wifi + 📱 cell) +
  обход разово, base64, no-store.
- **МОДЕЛЬ ОДНОЙ ПОДПИСКИ:** один источник Lastdep → Worker отдаёт оба набора в
  одном ответе; каждый сервер дважды (🛜/📱), имена различаются меткой (Loon не
  схлопывает). Разделение по сети — МЕТКОЙ; фильтры режут regex по 🛜/📱.
  Каждая функция — родитель `select` из двух fallback-детей (-W/-C); netwatch
  флипает родителя при смене сети (`setSelectPolicy`). Узел выбирает fallback сам.
- **AI — страновые тиеры, СТРОИТ Worker** динамически по числу узлов (DE первой;
  тай-брейк скорость→близость к DE; одиночные в catch-all; RU/BY исключены). В конфиге
  плейсхолдеры `# __RH_AI_FILTERS__` / `# __RH_AI_GROUPS__`.
- **Гист** `b14194982ad9d058c21d393d0f342147`: `lastdep-nodes.txt` (эталон, base64),
  `metrics-k1.json` (merge), `devices.json` (реестр+флаги). (`nodes-kN.txt` Worker
  больше НЕ пишет — можно удалить вручную.)
- **k1** (Диана) забиндан. **k2** свободен — выдать жене (см. `ЭТАП_D_ПРОГРЕСС.md`).
- **Скрипты (активны только два):** `routehub-speedtest.js` v0.4.13 (cron 20 мин),
  `routehub-netwatch.js` v0.4.0 (network-changed: флип -W↔-C по сети + детект
  whitelist + детект оператора через /whoami). **Удалены:** core, health, ai-bad,
  viewer (выбор узла теперь делает fallback, не скрипт).
- **Группы:** `RH-Все`, `RH-Обход`, `RH-Прямой` (select DIRECT/Обход — тумблер
  whitelist), `RH-Главный` (select RH-АВТО/DIRECT — FINAL), `RH-AI`+`RH-AI-W/-C`
  (строит Worker), `RH-АВТО`+`-W/-C`, `RH-Звонки`+`-W/-C`. (`RH-Apple` убрана —
  Apple идёт DIRECT.)
- **conf = C-draft-18** (живой). Ссылка: `…/config?key=k1`.

## ГЛАВНЫЕ ВЫВОДЫ (зафиксировано)

1. **Выбор узла делает fallback, не скрипт.** Скрипты-селекторы удалены; netwatch
   только флипает родительские select-группы по сети + детект whitelist/оператора.
2. **AI без логина = страна + стабильность.** Держим на Германии (тир DE первой).
   Проблема Gemini — репутация выходного IP под аккаунтом, не тип IP.
3. **СТРАНА УЗЛА — ИЗ ИМЕНИ** (флаг 🇩🇪 или слово), НЕ из GeoIP (`rating.country` врёт:
   финский узел с country=DE и т.п.). `countryFromName` в Worker/netwatch.
4. **Конфиг — одна ссылка** (Worker, keyed `?key=kN`). Токен GitHub только на Worker.
5. **`setSelectPolicy`** переключает select-группу (рабочий). fallback-группой скрипт
   не управляет. async/await в Loon (cron/network-changed/generic) работают.
6. **Whitelist детектит ТОЛЬКО netwatch на устройстве** (маяки generate_204 молчат +
   ya.ru жив). Worker (`*.workers.dev`) под whitelist недоступен.
7. **РФ→DIRECT по ИМЕНИ домена** (доменные правила бьют GEOIP и FINAL; ловят РФ-сервисы
   на зарубежных CDN) — так РФ-приложения не видят прокси.
8. **Банки под whitelist не гарантированы** (DIRECT режет whitelist, обход с зарубежным
   выходом режет геоблок банка). В обычном режиме все работают.
9. **TikTok — БЕЗ MITM** (маршрутизация доменов, Этап J). **Авторизованная проверка AI
   отвергнута** (бан).
10. **Приоритет правил Loon (3.9):** доменные раньше IP; локальные > плагин > подписка;
    далее сверху вниз; FINAL последний.
11. **fallback и DIRECT-член:** не подтверждено, тестит ли Loon DIRECT в fallback —
    ключевой открытый вопрос Этапа F (`ЭТАП_F_ЗАМЕТКИ.md`).
