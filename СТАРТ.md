## ГЛАВНЫЕ ВЫВОДЫ (зафиксировано)

1. **Выбор узла делает fallback, не скрипт.** Порядок узлов задаёт Worker по
   композитному баллу (down/med/jit/bl); netwatch только флипает родителей по сети.
2. **ОДНА ССЫЛКА НА ВСЕХ БЕЗ ?key НЕВОЗМОЖНА:** Loon не передаёт идентификатор
   устройства в запросе подписки (UA общий, IP меняется, cookie не хранятся), а
   метрики персональны. key=kN — идентификатор; ПРИВЯЗКА автоматическая (nonce
   при первом POST /speed), запасной ключ создаётся сам. Выдача устройства = та же
   ссылка с другим key.
3. **СТРАНА УЗЛА — ИЗ ИМЕНИ** (флаг/слово), НЕ из GeoIP (врёт).
4. **Пинг Loon ≠ наш rtt/med:** Loon — одиночный свежий замер через узел раз в 300с;
   мы — статистика из 3 проб (min=rtt, медиана=med, балл по med). Цель теперь одна
   (generate_204), но цифры один в один не совпадут по построению.
5. **`setSelectPolicy`** работает (select); fallback-группой скрипт не управляет.
6. **Скрипты живут только при активном VPN:** самоподнятие при выключенном Loon
   невозможно; разрывы видны постфактум в истории (rh_runlog). «Всегда включено»
   в Loon включено. $loon отдаёт только модель/версии — батареи в API нет.
7. **Whitelist детектит ТОЛЬКО netwatch на устройстве**; Worker под whitelist недоступен.
8. **РФ→DIRECT по ИМЕНИ домена** (доменные правила бьют GEOIP и FINAL).
9. **Приоритет правил Loon (3.9):** доменные раньше IP; локальные > плагин > подписка.
10. **Текст протокола под именем узла (vless+vision…) править НЕЛЬЗЯ** — рендер Loon
    из параметров узла; меняется только ИМЯ (метки через фрагмент URI).
11. **Один плагин на всё — не делаем:** правила плагина не ссылаются на именованные
    группы (И8); выигрыша над «одна ссылка = весь конфиг» нет.
12. **Региональные штрафы в балле — ОТКЛОНЕНЫ** (метрики уже отражают регион);
    точечные исключения — фильтром по флагу (как RU/BY в AI).
13. **fallback и DIRECT-член:** не подтверждено, тестит ли Loon DIRECT в fallback —
    ключевой открытый вопрос Этапа F.
